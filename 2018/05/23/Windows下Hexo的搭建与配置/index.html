Windows下 Hexo 的搭建与配置

发表于 2018-05-23 | 分类于 -教程
字数统计 | 阅读时长

一、安装Hexo

安装Git

  • 下载并安装Git
  • 不会Git的同学请学习Git的使用

安装Node.js

  • 下载并安装Node.js

在安装Node.js请勾选Add to PATH将Node.js添加到环境变量中

安装Cmder

  • 下载并安装Cmder
  • 将Cmder添加到环境变量下：右键我的电脑->属性->高级系统设置->环境变量->双击Path->将Cmder的安装路径添加进去
  • 打开Cmder输入Cmder.exe /REGISTER ALL
  • 完成上述步骤后，我们在任何地方右键都会有一个Cmder Here

    windows下我比较喜欢Cmder，也可以直接用Git bash

安装Hexo

  • 创建一个文件夹blog
  • Cmder Here
  • 使用npm安装Hexo：$ npm install -g hexo-cli

二、建站

_config.yml

网站的配置信息，可以在此配置大部分参数。

通常没有必要修改这一部分的内容

搭桥到github

  • 没有账号的创建一个账号
  • 创建一个New repository,如下图所示：

  • 仓库名为githubName.github.io,这里的仓库名必须为按照这样的规则来创建,然后Creat repository就好了

  • 进入刚才创建的仓库的setting里，找到GitHub Pages,在这里可以选择一个自己喜欢的主题使用。博主的主题叫next，后面会告诉大家怎么配置这个主题，实现跟博主一样的效果

  • 打开你的_config.yml,修改文件的一些配置
  • 进入你的blog目录，Cmder Here，执行以下命令,我们在浏览器里输入http://localhost:4000/，我们就能在本地服务器上看到我们的博客了
  • 接下来，我们要将我们之前做好的一切上传到Github上了
    • 上传之前，我们需要安装非常重要的东西npm install hexo-deployer-git --save
    • 然后我们就着手上传了

接下来，我们在浏览器打开githubName.github.io，我们就能看到属于我们自己的博客。没看到的朋友也不要着急，上传到github上也需要一点时间的，去打一盘游戏，回来就好了。我的下一篇会介绍主题的配置，主要是我的这个next主题的配置
