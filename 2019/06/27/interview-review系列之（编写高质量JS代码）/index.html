全局变量的问题

由于JS的两个特征，我们可以在不经意间创建出全局变量。

  • 不需要声明就可以使用变量
  • JS有隐含的全局概念，不使用声明的变量都会成为一个全局对象属性。

我们知道全局变量的产生有两种方式，隐式全局变量和明确定义的全局变量。但是它们产生的全局变量是不一样的

  • 通过var创建的全局变量是不能被delete操作符删除的。
  • 而隐式创建的全局变量是可以被delete操作符删除的。

通过这个可以说明，在技术上，隐式全局变量并不是真正的全局变量，它们是全局对象的属性，属性可以通过delete删除的，但是变量不可以。

单var形式

在函数顶部使用单var语句是比较有用的的一种形式，其好处在于

  • 提供了一个单一的地方去寻找所需要的局部变量
  • 防止变量在定义之前使用的逻辑错误

++ 和 –

还有一个对循环进行调整的是使用i += 1 / i = i + 1 / i -= 1 / i = i - 1来替代i++ / i--

这是因为在《JavaScript语言精粹》书中写到，++ --是鼓励了一种不够严谨的编程风格。大多数的缓冲区溢出错误所照成的安全漏洞，都是由像这样的编码照成的。使用++ --的时候，代码往往会变的拥挤、复杂和隐晦。

避免隐式类型转换

JS变量在比较的时候会隐式类型转换，所以false == 0是true，为了避免隐式类型转换，我们在比较值的时候最好使用===和!==操作符
