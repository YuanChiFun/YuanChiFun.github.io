VUE

使用步骤

引包

启动Vue

new Vue()

配置options对象

  • el:目的地；
    • template:模板；
    • data: function(){return {要使用的key:value}}

插值表达式

  • 对象
  • 字符串
  • 布尔值
  • 三元表达式

指令

以v-xxx表示

  • v-text : inner-text
  • v-html : innerHTML
  • v-if : 判断是否加入这个元素
  • v-else-if
  • v-else
  • v-show：隐藏元素，相当于给元素添加了display:none
  • v-bind:给元素属性赋值（单向）
    • v-bind:属性名 = “变量”
  • v-on:给元素添加事件
    • v-on:原生事件名 = “给变量进行操作||函数名”
  • v-model:双向数据流绑定
    • v-model = “变量名”
  • v-for
    • 数组：v-for = “item/(item,index) in Array”
    • 对象：v-for = “item/(item,key,index) in obj”
  • methods:这里的this就是data函数return出去的对象，这里的this是指向，不是事件对象。

组件渲染

  • components:声明要用的组件，key是组件名，value是组件对象。

注册全局组件

公共性组件，不必每次都声明

  • Vue.component(‘组件名’，组件对象)；

过滤器

  • filter||filters
    • 全局过滤器（给数据添油加醋显示）：Vue.filter(‘过滤器名’，过滤方式fn)；
    • 组件内：filters:(‘过滤器名’，过滤方式fn)；

监视

  • watch:监视一个数据

    复杂数据要深度监视

  • 使用：

包含原值不换，缓存不变的机制

slot

父组件传递的DOM结构，vue提供的内置组件

<slot name:""></slot>

组件生命周期

1.beforeCreate:组件创建之前

2.created:组件创建之后 ，可以操作数据，可以实现vue->页面的影响，应用：发起ajax请求

3.beforeMount:vue起作用，装载数据表到Dom之前 只执行一次

4.mounted: —————-之后

5.boforeUpdate: 基于数据改变影响页面，当有数据才会触发 获取原DOM

6.updated：获取新DOM

7.beforeDestory:销毁之前 对应父组件 v-if false

8.destoryed:销毁之后

9.activated:

10.deactivated:

<keep-alive></keep-alive>内置组件，如果有keep-alive，v-if控制的是组件的激活，没有的话控制的是组建的销毁

获取DOM元素

  • 在组件的DOM部分，任意标签中，写上ref=’xxx’;
  • 通过组件对象this.$ref.xxx，获取到元素
  • 希望Vue真正渲染DOM到页面以后，才做的事
    • this.$nextTick(function(){
      this.$refs.xxx.操作名})

SPA

  • SPA锚点值改变后，不会发起请求，发起ajax请求，局部改变页面数据
    • 页面不跳转，用户体验更好
  • haschange事件：url上的部分锚点数据(#xxx)改变，可以获取到这个事件

meta

  • 路由meta元数据
    • meta是对于路由规则是否需要验证权限配置
    • 路由对象中和name属性同级{meta:{isChecked:true}}
  • 路由钩子
    • 权限控制的函数执行时期
    • 每次路由匹配后，渲染到组件到touter-view之前
    • router.beforeEach(function(to,form,next){})
      • to：从哪里来
      • from：到哪里去
      • next：放行，不调用next就会卡住

        • next(false)不放行
        • next({name:’xxx’})重定向
